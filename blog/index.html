Programação Orientada a Objetos em TypeScript

Publicado em 11 de março de 2025

Este artigo explora os mecanismos da linguagem TypeScript que permitem a implementação da Programação Orientada a Objetos (POO), abordando os quatro pilares fundamentais: Herança, Polimorfismo, Encapsulamento e Abstração.

Mecanismos da Linguagem

Objeto

Em um contexto isolado, um objeto pode ser definido como um tipo de dado usado para armazenar uma coleção de valores, que podem ser dados primitivos ou outros objetos, organizados em pares chave/valor (key/value). No exemplo a seguir, o objeto pessoa armazena diversas informações, como a chave nome, que contém o valor “Lucas” do tipo “string” e a chave endereco que armazena outro objeto. 

const pessoa = {

          nome: "Lucas",
          sobrenome: "Garcez",
          idade: 28,
          endereco: {
          cidade: "Melbourne",
          pais: "Austrália",
        },
      };

Classes, Atributos e Métodos

Uma classe define a estrutura e o comportamento de um objeto através de atributos e métodos. 

class Pessoa {
            nome: string;
            sobrenome: string;
            idade: number;

            constructor(nome: string, sobrenome: string, idade: number) {
              this.nome = nome;
              this.sobrenome = sobrenome;
              this.idade = idade;
            }

            obterNomeCompleto() {
              return `${this.nome} ${this.sobrenome}`;
            }
          }

Instância

Uma instância é um objeto criado a partir de uma classe.

const lucas = new Pessoa("Lucas", "Garcez", 28);
console.log(lucas.obterNomeCompleto()); // "Lucas Garcez"
